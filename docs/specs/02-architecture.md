# 02 技术架构规范

## 文档元信息

- 版本：v1.0
- 状态：已冻结（MVP阶段）
- 最后更新：2026-05-08

## 总体架构

- 客户端框架：Tauri（Rust 后端 + Web 前端）
- 转换核心：Pandoc
- 可选预处理层：Node/Python（按需）
- 数据持久化：MVP 先 JSON，后续可迁移 SQLite

## 分层设计

1. UI 层（Web）
   - 页面：工作台、批量生成、历史记录、设置
   - 职责：参数采集、状态展示、错误提示

2. 应用服务层（Rust Commands）
   - 职责：任务编排、参数校验、调用转换引擎、落盘记录

3. 核心引擎层
   - 预处理器：文本清洗、结构归一化
   - 模板引擎：变量填充、模板解析
   - 转换引擎：Pandoc 调用与输出控制

4. 基础设施层
   - 文件系统访问
   - 日志系统
   - 任务历史存储

## 模块边界

- `converter`：纯转换逻辑，不关心 UI。
- `template`：模板加载/校验/变量注入。
- `history`：任务记录写入与查询。
- `logging`：统一日志与错误序列化。
- `commands`：面向前端的稳定接口。

## 架构约束

- 所有 UI 触发后端行为，必须通过 command contract。
- 转换与模板模块禁止直接依赖 UI。
- 所有错误必须转为统一错误结构返回。
