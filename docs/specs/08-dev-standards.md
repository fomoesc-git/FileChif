# 08 开发规范

## 文档元信息

- 版本：v1.0
- 状态：已冻结（MVP阶段）
- 最后更新：2026-05-08

## 分支与提交

- 分支：`feature/*`、`fix/*`、`chore/*`
- 提交信息：`type(scope): summary`

## 目录约定

- `src-tauri/`：Rust 后端
- `src/`：前端 UI
- `templates/`：模板资产
- `docs/specs/`：规范文档
- `docs/milestones/`：里程碑总结

## 代码规范

- Rust：优先小模块、强类型、错误显式传播。
- 前端：组件职责单一，状态集中管理。
- 禁止在 UI 层拼接 shell 命令。

## 测试策略（MVP）

- 单元测试：预处理、模板替换、参数校验。
- 集成测试：单文件转换闭环。
- 回归样本：固定 10 份输入文档。

## 文档协作规范

- 技术决策必须落文档。
- 改规范先改 `docs/specs/*`，再改实现。
- 每个里程碑完成后新增总结文档。

## 版本与变更记录

- v1.0（2026-05-08）：首版开发规范。
