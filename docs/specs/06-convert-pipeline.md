# 06 转换流程规范

## 文档元信息

- 版本：v1.0
- 状态：已冻结（MVP阶段）
- 最后更新：2026-05-08

## 标准流程

1. 输入接收
   - 文件路径或原始文本
2. 输入校验
   - 编码、文件存在性、大小限制
3. 预处理
   - TXT 段落归一化
   - AI 对话去噪（角色前缀、重复段）
   - 标题/列表修复
4. 模板装配
   - 注入变量
   - 应用模板元数据
5. Pandoc 转换
   - 生成 DOCX/PDF
6. 结果落盘
   - 输出文件
   - 历史记录
   - 日志

## 处理约束

- 单任务默认超时：60 秒。
- 超时或失败必须回写 history 失败记录。
- 输出目录不可写时，返回 `OUTPUT_DIR_NOT_WRITABLE`。
- 当前 MVP 输入限定为 Markdown 文件路径，支持 `.md` 与 `.markdown`。
- 当前 MVP 输出扩展名必须与命令格式一致：`.docx` 或 `.pdf`。

## 观察指标

- `conversion_duration_ms`
- `input_size_bytes`
- `output_file_count`
- `error_code`
